## 📥 需求收集

### 如何提交需求
1. 创建GitHub Issue
2. 使用需求模板
3. 标注类型和优先级

### 需求模板
```markdown
## 需求标题

**类型**: 功能/Bug/优化/文档
**优先级**: P0/P1/P2/P3
**来源**: 用户反馈/内部/竞品分析
**提出者**: @username
**日期**: YYYY-MM-DD

### 问题/机会
描述当前的问题或机会

### 解决方案
建议的解决方案

### 价值
- 用户价值: ...
- 业务价值: ...

### 工作量
- 估计: X小时/天
- 难度: 低/中/高

### 相关链接
- 竞品参考: 
- 设计稿:
- 相关Issue:
```

## 🔄 需求评审流程

### 每周评审 (周日)
1. 回顾本周完成的需求
2. 评估未完成的需求
3. 调整下周优先级
4. 收集新需求

### 评审标准
- 用户价值: 解决什么问题?
- 业务价值: 带来什么收益?
- 技术可行性: 能否实现?
- 资源投入: 需要多少成本?
