# PowerClaw CEO 每日管理框架

## 🎯 核心原则
- **永不停止**: 每天必须有产出
- **PDCA循环**: 计划(Plan) → 执行(Do) → 检查(Check) → 优化(Act)
- **双线并行**: 内容运营 + 产品研发

---

## 📅 每日流程 (Daily Routine)

### 晨会 (09:00-09:30)
1. **检查昨日状态**
   - 阅读 worklog/daily-*.md
   - 检查GitHub提交记录
   - 查看内容发布状态

2. **制定今日目标**
   - 内容目标: 1篇文章/天
   - 产研目标: 1个功能/3天
   - 运营目标: 3条社媒/天

3. **分配任务**
   - 内容组: 选题 → 创作 → 审核 → 发布
   - 产研组: 需求 → 设计 → 开发 → 测试 → 发布
   - 运营组: 发布 → 推广 → 互动 → 数据

### 执行时段 (09:30-18:00)
- **上午**: 内容创作 + 深度工作
- **下午**: 产研开发 + 协作沟通
- **碎片时间**: 社媒运营 + 社区互动

### 晚会 (18:00-18:30)
1. **今日检查**
   - 目标完成度?
   - 遇到什么阻塞?
   - 需要什么资源?

2. **明日计划**
   - 确定明日优先级
   - 预留缓冲时间

3. **日志记录**
   - 更新 worklog/daily-YYYY-MM-DD.md
   - 更新内容日历

---

## 📊 双轨工作制

### 轨道A: 内容运营 (日常维护)
**频率**: 每天
**产出**: 
- 1篇博客文章
- 3-5条社交媒体内容
- 1次社区互动

**流程**:
```
选题 → 研究 → 大纲 → 创作 → 审核 → 发布 → 推广 → 数据
```

### 轨道B: 产品研发 (持续升级)
**频率**: 迭代制
**产出**:
- 每周1-2个功能发布
- 持续bug修复
- 用户体验优化

**流程**:
```
需求收集 → 优先级排序 → 设计 → 开发 → 测试 → 发布 → 反馈
```
